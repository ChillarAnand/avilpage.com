FASTQ has become the de facto standard for storing the output of high throughput sequencing instruments such as the Illumina Genome Analyzer. So, it is necessary for bioinformatics scientists to zip/unzip, read & process these files on daily basis. This is a simple tutorial to get started working with those files.

1. Understand Structure Of FASTQ File.

FASTQ format is a text-based format for storing both a biological sequence (usually nucleotide sequence) and its corresponding quality scores. Both the sequence letter and quality score are encoded with a single ASCII character for brevity. A FASTQ file normally uses four lines per sequence.

Lets look at first 4 lines of sample FASTQ file.


  • Line 1 begins with a '@' character and is followed by a sequence identifier and an optional description (like a FASTA title line). 
  • Line 2 is the raw sequence letters. 
  • Line 3 begins with a '+' character and is optionally followed by the same sequence identifier (and any description) again. 
  • Line 4 encodes the quality values for the sequence in Line 2, and must contain the same number of symbols as letters in the sequence.

2. Find Number Of Lines.

Usually the size of FASTQ file will be few GB and most of them will be compressed. So, it is difficult to extract such a file. But don't worry, Linux can handle those files quite easily without even extracting. To find the number of lines in a file just use 
 
                                          bzcat filename | wc -l

to get number of lines in the file. Divinding that by 4 will give you number of reads in that file.

3. Split File Into Chunks:

Most of the times it is necessary to split the huge file into several chunks.
                                   
                                  split -l 400000 sample.fastq sample_part

will split the the file into several parts each containing 400,000 lines and their names start with parts.



4. Unzipping The File: 

Also sometimes it is necessary to unzip the file for processing and analyzing. But if you normal unzip commands it will throw an error. You should use


                      zcat myreads.fastq.gz | split -d -l 2000000 - Block
