Every web application has to serve static files like JavaScript, CSS, images and media files which are uploaded by the user or files which user can download.

Serving Files In Development:

Store all your static files in a directory called static at project root. If have any static files related to a single app, you can create a static directory inside your app and you can store there. You have to specify all such directories in your settings file. Also specify an url for static files.
STATICFILES_DIRS = (
    os.path.join(BASE_DIR, "static"),
    os.path.join(BASE_DIR, "some/other/dir/"),
)

STATIC_URL = '/static/'

If you have django.contrib.staticfiles in your INSTALLED_APPS & DEBUG = True in your settings, then runserver command will automatically serves these static files for you.

Now you can use these static files like these
#hardcoding urls
<img src="/static/images/sample.png" />
#or better use static template tag
{% load staticfiles %}
<img src="{% static "images/sample.png" %}" />

To serve media files, create new directory called media at your project root and place all your files in there. Next specify path so that django knows where the files are.
MEDIA_ROOT = join(BASE_DIR, 'media')
MEDIA_URL = '/media/'
But Django doesn't servers these files automatically. You have to specify these in project urls.
from django.conf import settings
from django.conf.urls.static import static

urlpatterns = [
    # your urls
] + static(settings.MEDIA_URL, document_root=settings.MEDIA_ROOT)

Now you can use these media files like these
#hardcoding urls
<img src="/media/music/sample.mp3" />
#or better use 
<img src="{{ MEDIA_URL }}music/sample.mp3" %}" />



Serving Files In Production:

Above settings are suitable for development purposes. In production we have to serve through some server. Before that lets collect all our static files into a single directory. Set STATIC_ROOT to the directory where you want all your files to be collected. Then run python manage.py collectstatic which copies all your static folders into STATIC_ROOT.

Now we have all our static files & media files
